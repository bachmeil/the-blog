Here's a description of my LLM workflow for programming. There's no intention to claim it's right or wrong, or that anyone else should be using it. Everyone should find what works best for them and then use it as a tool to make themselves more productive.



And here's the philosophy that drives my chosen workflow.

- AI completing a program is better than writing a program entirely.
- It feels better to design the program and understand all the parts. Cheaper too.
- I don't want to outsource all maintenance.
- I want to use AI as an assistant to do the things I don't want to do. I want to spend my time doing things I enjoy. Spending all day telling a machine what to do and then reviewing mostly black-box code would be awful.
